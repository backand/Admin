Parameters - Permissions

Durados allows to determine the permissions for each user role, for each view independently.

  • Permiossions can be granted or denied for different actions - add, edit, delete & read.

  • Override inheritable:
    • When this parameter is unchecked, the permissions would be inherited from higher level. No change on permissions would be possible on that level.
    • When this parameter is checked, the permissions grantted on that view override the permissions grantted by higher level. 

  • Grant Permissions:
    • Permissions can be granted to the following actions:
      • Create - allow adding or duplicating new row.
      • Edit - allow editing existing row.
      • Delete - allow delete existing row.
      • Read - allow looking at the view on read-only status.
    • Default situation on views is when no roles are defined at all. That status detemine that all users can view and edit the data = all granted.
    • When role has been defined as granted permission, the permissions are granted to this role only - all other roles loose their permissions, which means that they react as they were denied.
  • Deny Permissions:
    • Permissions can be denied to the following actions:
      • Create - deny the ability to add or duplicate new row.
      • Edit - deny the ability to edit existing row.
      • Delete - deny the ability to delete existing row.
      • Read - deny the ability to open the view, even on read-only status.
    • When role has been defined as granted permission, the permissions are granted to this role only - all other roles loose their permissions, which means that they react as they were denied.

  • Default situation on views is when no roles are defined at all. That status detemine that all users can view and edit the data = all granted.

  • Few "tips" when defining permissions:
    • Delete action is the most problematic. Therefore it should receive the most restricted permissions, in order to narrow the cases to delete by mistake.
    • View permissions are mostly granted to all users, because mostly no harm happens by viewing the data.
