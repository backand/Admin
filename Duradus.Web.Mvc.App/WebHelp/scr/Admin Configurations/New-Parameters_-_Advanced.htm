Parameters - Advanced

In case a new view have been created, there are advanced parameters that can be in use for ad view:min users, in order to improve the appeareance and behaviour of the view:

  • Durados Buttons:
    • Promote Button Name: defines the name of the promote button.
      Default name is Promote.

    • Duplicate Button Name: defines the name of the duplicate button.
      Default name is Duplicate.

    • New Button Name: defines the name of the new button.
      Default name is New.

    • Insert Button Name: defines the name of the insert button.
      Default name is Insert.

    • Edit Button Name: defines the name of the edit button.
      Default name is Edit.

    • Delete Button Name: defines the name of the delete button.
      Default name is Delete.

    • Add Items Field Name: defines the name of add items button.
      There is no default name.

  • Database columns names - display the default names that will be given the following columns on DB:

    • Create Date Column Name: defines the name of the Created date column.
      Default name is CreateDate.

    • Delete Button Name: defines the name of the delete button.
      Default name is Delete.

    • Delete Button Name: defines the name of the delete button.
      Default name is Delete.

    • Delete Button Name: defines the name of the delete button.
      Default name is Delete.

  • Workflow Parameters - In case that the view contain a workflow, there are several parameters that can defined.
    Explaination on those parameters can be found under Workflow Parameters

  • Design Parameters -  
    • Container Graphic - The Container Graphic stands for the CSS class in the view.
    • Row Color Column Name -
      • A field containing the Highlight option can be added to the view - for now the name should be RowColor.
      •  Entering the field name ton that parameter will allow to use the highlight option.

         

  • There are several more parameters on Advanced category:
    • Base View:

    • Permanent Filter:
      • There are cases when the view should not display all the data it contains.
      • In case that some of the data should be always removed from display, a permanent filter can determine what data would be left out
      • The format of the condition should be: '<Name> <condition> <value>'(Name - represnt the field name, as appear on the Database).
        • For example, if on a specific view all the data earllier then 1/1/2000 is irrelevant, a permanent filter can be established as following:
        • 'Date >= 1/1/2000' - The view would present only data from the requested date forward.
      • The following can be performed on any column, of any kind.
      • Note that unlike Default Filter, there is no option to display the filtered out data on the view, unless the filter would be removed. 
      • Permanent filter can also be established for several columns:
        •  'Date >= 1/1/2000 and Status = 'Open'' - all rows that are earlier then this date or in status Open would not appear on the view.

    • Show Up Down: Display the Up & Down parameters on table, that allows to change the order of rows, in case they are sorted  by Ordinal column.
